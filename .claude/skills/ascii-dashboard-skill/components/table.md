## Spec Template
```json
{
  "id": "(required) string вҖ” unique snake_case identifier",
  "type": "table",
  "title": "(required) string вҖ” display name",
  "category_field": "(required) string вҖ” primary dimension field",
  "value_field": "(required) string вҖ” primary measure field",
  "columns": [
    {
      "field": "(required) string вҖ” field name",
      "label": "(optional) string вҖ” column header display name",
      "format": "(optional) number_comma | percentage | currency | decimal"
    }
  ],
  "sort_by": "(optional) string вҖ” field name to sort by",
  "sort_order": "(optional) asc | desc вҖ” default: desc",
  "variant": "(optional) standard | highlight вҖ” default: standard",
  "grid": {
    "col": "(required) int 1-12",
    "row": "(required) int",
    "width": "(required) int вҖ” typically 8-12",
    "height": "(required) int вҖ” typically 2-3"
  }
}
```

## Sizing Guidelines
- **Width**: 8-12 columns (use 8 for side-by-side with a chart, 12 for standalone)
- **Height**: 2-3 rows (use 3 for tables with many rows)
- **Common pattern**: Full-width table (12 cols) at dashboard bottom, or paired with pie chart (8+4)
- **Best practice**: Keep columns to 3-6 for readability; use `sort_by` to highlight top/bottom items
